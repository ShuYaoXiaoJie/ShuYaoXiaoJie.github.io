wampserver登录mysql出现问题

原因：可能是mysql用户名和密码与phpMyAdmin的配置文件的用户名密码不对应
解决办法：1、打开MySQL目录下的my.ini文件，在文件的最后添加一行“skip-grant-tables”，保存并关闭文件。
2、重启MySQL服务。
3、在命令行中输入“mysql -uroot -p”(输入密码)，回车即可进入数据库。
4、执行，“use mysql;”使用mysql数据库。
可以用cmd控制台登陆也可以用wamp的mysql控制台mysql console登陆
->use mysql
->update user set password=password(“密码”) where user=’root’;
5.修改phpMyAdmin的配置文件的用户名密码
\wamp\apps\phpmyadmin3.4.10.1下的config.inc.php文件
$cfg[‘Servers’][$i][‘user’] = ‘root’;
$cfg[‘Servers’][$i][‘password’] = ‘密码’;
6.这一步很重要，关系到你半天的努力。我就是因为这一步反复弄。
重启服务之后，重新登陆phpMyAdmin
然后出现未修改的页面#1045 - Access denied for user ‘root’@’localhost’ (using password: YES)
然后最后一步就是刷新页面（注：不是重新打开phpMyAdmin而是刷新错误的页面，一般就好了）

wampserver多站点配置

1、在D:\wamp\bin\apache\apache2.4.9\conf\extra下找到httpd-vhosts.conf文件，在最下方添加站点


DocumentRoot “E:/Demo/test01”
ServerName test01.com
DocumentRoot “E:/Demo/test02”
ServerName test02.com


DocumentRoot “E:/Demo/“
ServerName localhost

2、在httpd.conf文件中去掉 Include conf/extra/httpd-vhosts.conf前面的#，
（我试了几次的版本房中不去掉#也没有什么关系）
3、在E:\Demo\test01\下面创建index.php文件进行测试，test02文件下同理
4、以管理员身份打开C:\Windows\System32\drivers\etc下的hosts文件，在最下方添加站点：
127.0.0.1 test01.com
127.0.0.1 test02.com
127.0.0.1 localhost

更改wampserver的网页访问位置

1、打开WAMPSERVER下Apache下的httpd.conf文件
2、ctrl+f快捷键搜索documentroot，找到
#
DocumentRoot “D:/wamp/www/“，改成新网页项目的目录。
3、往下找到“
” ，更改地址。
4、重启所有服务。
5、注意：最好用除记事本以外的文本编辑器打开，我试了好几遍用记事本打开并修改重启后会显示黄色图标。用sublime打开修改即可。还有地址的格式要与之前配置文件给出的地址格式保持一致，即：E:/学习/代码/PHP/Demo/
6、修改所有的配置文件都需要重启才能生效。
7、test.php文件不能存放在中文目录下，不然会出现乱码。我的存放位置：E:\Demo

自定义wampserver网站根目录

1、E盘下新建Demo文件夹
2、将DocumentRoot 下的路径改为”E:/Demo/“
3、将下的路径也改为”E:/Demo/“
4、找到D:\wamp下的wampmanager.ini文件，搜索menu.left,将 Caption: “Demo 目录”; Action: shellexecute; FileName: “E:/Demo”;更改如此。
5、找到D:\wamp下的wampmanager.tpl文件，将Type: item; Caption: “${w_wwwDirectory}”; Action: shellexecute; FileName: “${wwwDir}”; Glyph: 2改为
Type: item; Caption: “Demo 目录”; Action: shellexecute; FileName: “E:/Demo/“; Glyph: 2
6、退出重启即可。
